# OCR технических требований на PDF-чертежах КОМПАС

Практическая работа по извлечению технических требований из инженерных PDF-чертежей.

Основная задача: найти на PDF-чертежах КОМПАС блок технических требований, извлечь текст, определить координаты блока и сохранить debug-изображение с обведённой найденной областью.

## Что реализовано

В проекте реализован pipeline:

PDF КОМПАС с текстовым слоем → извлечение текста и координат → поиск блока технических требований → проверка качества текста → JSON/debug

Основной скрипт:

`experiments/check_pdf_kompas_textlayer.py`

Скрипт проверки качества текста:

`experiments/text_quality_checker.py`

Запасной OCR-вариант для PDF без текстового слоя:

`experiments/check_pdf_v5_post.py`

## Почему используется текстовый слой PDF

Новые тестовые чертежи являются PDF-чертежами КОМПАС с текстовым слоем. Поэтому основной способ обработки — не OCR по картинке, а извлечение текста и координат напрямую из PDF.

OCR через Tesseract оставлен как запасной вариант для PDF без текстового слоя.

## Что выдаёт программа

После обработки создаются:

- `output.json` — координаты найденного блока и извлечённый текст;
- `output_quality.json` — текст после проверки качества, исправления и предупреждения;
- `report.txt` — человекочитаемый отчёт по качеству текста;
- `debug_selected/` — debug-картинки с выбранным блоком;
- `debug_all_candidates/` — debug-картинки со всеми кандидатами.

## Установка на Ubuntu / Debian / Astra Linux

Системные зависимости:

`sudo apt update`

`sudo apt install -y python3-venv poppler-utils tesseract-ocr tesseract-ocr-rus libgl1 libglib2.0-0`

Python-окружение:

`python3 -m venv .venv`

`source .venv/bin/activate`

`pip install -r requirements.txt`

## Запуск для PDF-чертежей КОМПАС

Пример запуска основного извлечения:

`python experiments/check_pdf_kompas_textlayer.py test_server/mock_files/kompas_pdf --out results/final_kompas/extract --dpi 180 --max-blocks 2`

Проверка качества извлечённого текста:

`python experiments/text_quality_checker.py --input results/final_kompas/extract/output.json --out results/final_kompas/text_quality`

Главный итоговый файл:

`results/final_kompas/text_quality/output_quality.json`

## Результат на тестовом наборе

На тестовом наборе из 5 PDF-чертежей КОМПАС с текстовым слоем программа нашла блок технических требований во всех 5 файлах.

Для каждого файла был найден один блок технических требований, сохранены координаты, извлечён текст и создана debug-картинка.

Проверка качества текста дополнительно выявляет типовые проблемы PDF-текстового слоя:

- латинское `mK` вместо кириллического `мК`;
- точку вместо запятой в значениях типа `0.4 мм`;
- пропущенный пробел в `Гр.I`;
- подозрительные слова, которые требуют ручной проверки.

## Ограничения

Проект является учебным/исследовательским baseline, а не промышленной системой.

Алгоритм проверен на небольшом наборе PDF-чертежей КОМПАС с текстовым слоем. Для других типов PDF, сканов, AutoCAD-PDF или чертежей без текстового слоя может потребоваться OCR fallback и дополнительная настройка.
