# OCR / text-layer эксперименты для PDF-чертежей

В этой папке находятся экспериментальные скрипты для поиска блока технических требований на инженерных PDF-чертежах.

## Основные файлы

`check_pdf_kompas_textlayer.py` — основной рабочий скрипт для PDF-чертежей КОМПАС с текстовым слоем.

`text_quality_checker.py` — проверка качества извлечённого текста.

`check_pdf_v5_post.py` — запасной OCR-вариант для PDF без текстового слоя.

## Основной pipeline для КОМПАС-PDF

PDF с текстовым слоем → слова и координаты → строки → блок технических требований → JSON/debug → проверка качества текста

## Что делает check_pdf_kompas_textlayer.py

Скрипт:

1. Открывает PDF через PyMuPDF.
2. Извлекает слова и координаты из текстового слоя.
3. Собирает строки.
4. Находит блок технических требований по пунктам 1, 2, 3, 4...
5. Учитывает строки-продолжения и подпункты через дефис.
6. Отбрасывает ложные элементы вроде размеров, фасок, радиусов и штампа.
7. Сохраняет координаты блока и текст в JSON.
8. Создаёт debug-картинки.

## Что делает text_quality_checker.py

Скрипт проверяет извлечённый текст и создаёт:

- corrected_text;
- confidence;
- warnings;
- review_required;
- correction_log.

Проверяются типовые ошибки:

- `mK` вместо `мК`;
- `0.4 мм` вместо `0,4 мм`;
- `Гр.I` вместо `Гр. I`;
- подозрительные слова;
- смешение латиницы и кириллицы;
- попадание служебных строк штампа.

## Проверенный результат

На тестовом наборе из 5 КОМПАС-PDF с текстовым слоем блок технических требований был найден во всех 5 файлах.

## Ограничения

Для PDF без текстового слоя используется OCR fallback `check_pdf_v5_post.py`.

Для промышленных условий нужен более широкий тестовый набор и дополнительная проверка на разных форматах чертежей.
