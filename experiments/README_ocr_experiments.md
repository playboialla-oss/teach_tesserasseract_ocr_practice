# Эксперименты OCR для PDF-чертежей

В этой папке находятся экспериментальные скрипты для поиска блока технических требований / текстовых примечаний на инженерных PDF-чертежах.

Основная текущая версия:

`experiments/check_pdf_v5_post.py`

## Цель

Скрипт должен обработать PDF-чертёж и получить:

- координаты найденного блока технических требований;
- распознанный текст;
- JSON-файл с результатами;
- debug-картинки с обведёнными найденными областями.

Общий пайплайн:

PDF → изображение → OCR → текстовые кандидаты → оценка кандидатов → выбранный блок → JSON/debug

## Что делает скрипт

Скрипт:

1. Берёт PDF-файл или папку с PDF.
2. Рендерит первую страницу PDF в изображение.
3. Запускает OCR через Tesseract.
4. Получает текстовые строки и координаты.
5. Собирает кандидаты на блок технических требований.
6. Оценивает кандидаты по признакам:
   - наличие слов ГОСТ, ОСТ, ТУ, РД;
   - наличие слов Материал, Покрытие, Маркировать, Допуски, Контроль;
   - наличие пунктов 1, 2, 3, 4;
   - похожесть на таблицу;
   - похожесть на размерную надпись;
   - положение блока на листе.
7. Сохраняет результат в JSON.
8. Сохраняет debug-картинки.

## Текущий статус

`check_pdf_v5_post.py` — лучшая текущая экспериментальная версия.

Она чаще всего находит область с техническими требованиями, но пока не является полностью готовым промышленным решением.

На тестовом наборе DWG-PDF чертежей остаются проблемы:

- иногда блок требований найден не полностью;
- иногда вместе с требованиями выбираются лишние размеры или графические элементы;
- иногда требования разбиты на несколько частей;
- иногда OCR ошибается в тексте.

## Как запустить

Из корня проекта:

`cd ~/teach_tesserasseract`

`source .venv/bin/activate`

Запуск на папке с PDF-чертежами:

`python experiments/check_pdf_v5_post.py test_server/mock_files/drawings --out results/check_pdf_v5_post --dpi 200 --max-blocks 3 --psm 11`

После запуска появятся:

`results/check_pdf_v5_post/output.json`

`results/check_pdf_v5_post/debug_selected/`

`results/check_pdf_v5_post/debug_all_candidates/`

`results/check_pdf_v5_post/debug_rejected/`

## Ограничения

Эксперименты проверялись на небольшом наборе DWG-PDF инженерных чертежей.

Для PDF, экспортированных из КОМПАС/ASCON, нужна отдельная проверка.

Tesseract используется как OCR-движок. Он распознаёт текст и даёт координаты, но сам не понимает, где находятся технические требования. Поэтому выбор нужного блока реализован отдельно в Python через правила, фильтрацию и анализ расположения текста.
